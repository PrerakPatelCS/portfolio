Simple Plagiarism Checker

A multithreading project in C that compares how similar 2 files are to each other. Takes in files or directories and uses threading to process everything, then will use threading to analyze each file and compare each file to each other. The Analysis is based on the Word Frequency Distribution and using that for the Jensen Shannon Distance formula to measure the similarity between the two probability distributions. The lower the number the more simialr the two files are.

EERTREE Visualization

The EERTREE is a palindromic tree that processes a strings efficiently for a palindrome, first introduced in a research paper in 2015. Each node is a palindrome, the edges are the characters you add to both sides, and the links point to the biggest palindrome that is a suffix to this palindrome. The way the site works is that when you start typing the characters get fed into a data stream I made which adds the characters to the EERTREE.

Matrix Multiplication

There are many different algorithms that have a faster runtime than naive matrix multiplication. I use the strategy pattern to test all the algorithms easily. The fastest algorithm's runtime is O(n^2.3) but it is just theory, Strassen's algorithm which is O(n^2.81) is the only one you can implement. Because it is divide and conquer, there is a lot of over head and the algorithms base case is a 2x2 matrix so the matrices may need some padding. There are 3 ways to add padding, convert the matrix to a power of 2 square matrix, dynmaic padding, and dynamic peeling. Another algorithm is Strassen Winograd which is a small optimization to Strassen. Interesting fact about quick select median of medians the run time complexity is O(n) but it is slower than normal quick select.

Books

Competitive Programmer's Handbook

This book goes over various data strucutres and algorithms and how to implement them C++. Shows you a bunch of patterns in certain problems and the algorithms that can tackle them, plus the time and space complexities of them. The implementation details focus on how to implement it with the least amount of code and goes over some tricks like using negative values and the default natural ordering instead of making a comparator. There are still some advanced algorithms I haven't gotten the chance to implement because of the rarity of those problems.

Effective Java

This book is organized with items which you can read in any order. The items go over the best practices for writing clean code in Java and goes over a bunch of idioms which can help you in any language you want. There is a lot of influence from Design Patterns and the JLS(Java Language Specification). The reason Design Patterns is important for Java is that the whole Collections Library and JDK is litered with these Design Patterns so when using them you understand how it all works and what their design intentions where.

A big reason why I wanted to read this book was because my friends usually solve problems in python and they are experts at python at this point, so I became the expert at Java and now my code has become very clean and I can help them out when they use Java. Also my Java code has become pythonic enough that sometimes my code would be cleaner than their python code which is very hard if you think about all the boilerplate!

Design Patterns: Elements of Reusable Object-Oriented Software

This book does not need an introduction. It goes over 23 design patterns for Object Oriented Programming. For each pattern it tells you the problem it solves, the consequences, the implementation, and all the parts that go into it like different interfaces and classes. It also goes over different implementations for different needs. The implementation is in C++ and Smalltalk which is obsolete as far as I know. The class diagrams are in OMT which is also obsolete but UML is based on OMT so it is not hard to understand. This book does a great job of accumulating experience in OOP and simplifying it down to 23 patterns for the readers. The book came out in 1995 and is still relevant today!

Interests

Competitive Programing

Competitive Programming has a series of contests where you are given around 4 to 8 problems and you have to solve them in a time constraint of around 2 hours. There are a bunch of different levels to Competitive Programming, Codeforces uses a scale similar to chess, an elo system. I was first introduced to this from my friends, at first we would do Leetcode questions and critque each others soltuions and then we got really good after a ton of practice. The natural progression is to move into Codeforces which is much harder than Leetcode. Codeforces is so hard that a lot of researchers use those questions and infrastructure to test out new algorithms or data structures. The new AI's are also tested on Codeforces to see how many questions it can solve.

Advent of Code

This was a fun event, similar to an Advent calender for Christmas you get a new puzzle every day from December 1st to 25th. After you complete the first puzzle there will be a second harder puzzle. Around 200 thousand people participate in this and the topics range from many different fields in computer science. This is a good way to challenge yourself, many people do it to learn a new language. I did it to get better at Python, I learned new techniques for coding in Python and with using a dynamic typing language. I also learned some new algorithms like Shoelace + Pick's theorems for finding area of a polygon. Check out my github repo with all my solutions.
